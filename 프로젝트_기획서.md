1. # 프로젝트 개요

1.1 서비스 설명

OSS_Poket은 포켓몬 정보를 검색하고 3D 모델로 확인할 수 있으며, 사용자가 좋아하는 포켓몬을 수집하고 관리할 수 있는 웹 애플리케이션입니다.

1.2 조원 정보

| 조원 이름 |
| --------- |
| 김우현    |
| 이건우    |

1.3 서비스 제목 및 주요 내용

서비스 제목: OSS_Poket - 포켓몬 3D 도감 및 컬렉션 서비스

주요 메뉴 구성:

| 번호 | 페이지명           | 경로         | 설명                             |
| ---- | ------------------ | ------------ | -------------------------------- |
| 1    | 온보딩 페이지      | /            | 서비스 소개 및 시작 화면         |
| 2    | 로그인 페이지      | /login       | Google OAuth를 통한 사용자 인증  |
| 3    | 포켓몬 도감        | /home        | 포켓몬 목록 조회, 검색, 필터링   |
| 4    | 포켓몬 상세 페이지 | /pokemon/:id | 포켓몬 상세 정보 및 3D 모델 뷰어 |
| 5    | 내 포켓몬 컬렉션   | /my-pokemon  | 사용자가 저장한 포켓몬 목록 관리 |
| 6    | 장바구니           | /cart        | 저장할 포켓몬 임시 보관          |

서비스 대상:

- 포켓몬 팬 및 수집가
- 포켓몬 정보를 찾는 사용자
- 3D 모델을 통해 포켓몬을 시각적으로 확인하고 싶은 사용자

  1.4 CRUD 데이터 셋

UserPokemon 모델 (최소 7개 이상의 필드 포함):

| 필드명     | 타입            | 설명                                    |
| ---------- | --------------- | --------------------------------------- |
| id         | Int (PK)        | 고유 식별자 (자동 증가)                 |
| user_id    | String          | 사용자 ID (Google OAuth)                |
| pokemon_id | Int             | 포켓몬 번호 (1~151)                     |
| added_at   | DateTime        | 포켓몬 추가 일시                        |
| user       | User (Relation) | 사용자 정보 (이메일, 이름, 프로필 사진) |
| created_at | DateTime        | 레코드 생성 일시                        |
| updated_at | DateTime        | 레코드 수정 일시                        |

추가 관련 데이터:

| 모델명       | 주요 필드                                                             |
| ------------ | --------------------------------------------------------------------- |
| User 모델    | user_id, email, name, picture, created_at, updated_at                 |
| PokemonModel | pokemon_id, model_path, cdn_url, model_type, file_size, storage_type, |
|              | is_primary, file_exists                                               |

5.1 포켓몬 관련 API

| 메서드 | 엔드포인트                                         | 설명                      | 요청/응답 예시                                                 |
| ------ | -------------------------------------------------- | ------------------------- | -------------------------------------------------------------- |
| GET    | /api/pokemon?userId={userId}                       | 사용자의 포켓몬 목록 조회 | 응답: { pokemon: [{ pokemon_id, added_at }] }                  |
| POST   | /api/pokemon                                       | 포켓몬 추가               | 요청: { userId, pokemonId, email, name, picture }              |
|        |                                                    |                           | 응답: { success: true, message: "포켓몬이 추가되었습니다." }   |
| DELETE | /api/pokemon?userId={userId}&pokemonId={pokemonId} | 포켓몬 삭제               | 응답: { success: true, message: "포켓몬이 삭제되었습니다." }   |
| GET    | /api/pokemon?id={pokemonId}                        | 포켓몬 모델 정보 조회     | 응답: { pokemon_id, model_path, model_type, url, file_exists } |
